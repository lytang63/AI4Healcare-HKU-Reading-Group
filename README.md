# AI4Healcare@HKU Reading Group

Public weekly reading group for **AI4Healcare@HKU**.  
We keep a lightweight, GitHub-first archive: **one row per week + one folder per session**.

## Start here
- How to add a session: **[tutorial.md](tutorial.md)**
- Notes template: **[template.md](template.md)**
- Weekly materials live under: `sessions/<term>/...`

## Formats (Weekly Topics)
1. **Completed work** — full project presentation
2. **WIP** — work in progress + challenges + feedback request
3. **Paper sharing**
4. **New model/tool exploration** / technical experience
5. **Conference experience sharing**
6. **Other valuable topics**

### Column guidelines
- **Materials**: link to session `[folder](...)`, or `[links](.../links.md)`, or external URLs
- **Recording**: optional; include link + access note if needed
- **Tags**: short, searchable hashtags (e.g., `#imaging #ehr #llm #benchmark`)

## Folder convention
- Term folders: `sessions/2026-spring/`, `sessions/2026-fall/`, `sessions/2027-spring/`, ...
- Session folder name: `YYYY-MM-DD-<type>-<short-title>/`
- `<type>` in: `completed | wip | paper | tool | conf | other`
- `<short-title>`: lowercase + hyphen (keep it short)

Example:
- `sessions/2026-spring/2026-03-04-wip-ehr-phenotyping/`

## External link policy (for public repo)
- Use “Anyone with the link can view” for Google Docs/Slides/Notion, etc.
- Avoid internal-only links unless clearly marked (e.g., “HKU login required”)
- Do NOT upload sensitive slides/recordings to a public repo; use a redacted version or omit
